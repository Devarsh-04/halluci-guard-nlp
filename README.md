# HalluciGuard v5  
### Retrieval-Augmented Hallucination Verification for Australian Visa Question Answering

HalluciGuard v5 is an NLP research project focused on detecting hallucinated or factually unsupported responses generated by large language models in a domain-specific Australian visa question answering system.

The project combines retrieval-augmented generation (RAG), semantic verification, natural language inference (NLI), and named entity consistency checking to identify whether generated answers are grounded in retrieved evidence.

This project was developed as part of an NLP coursework assessment at the University of Technology Sydney (UTS).

---

# Project Overview

Large language models often generate responses that appear fluent and convincing but may contain fabricated facts or unsupported claims. In high-stakes domains such as immigration guidance, incorrect information can lead to serious consequences for users.

HalluciGuard v5 addresses this problem through a lightweight multi-stage verification pipeline that:

- retrieves relevant evidence passages,
- generates answers using a language model,
- verifies factual consistency using multiple semantic signals,
- flags hallucinated responses.

---

# System Architecture

The pipeline consists of five major stages:

1. Query Processing  
2. Retrieval using Sentence-BERT + FAISS  
3. Cross-Encoder Re-ranking  
4. Answer Generation using Flan-T5-base  
5. Multi-Signal Hallucination Verification

Verification combines:
- Named Entity Recognition (NER)
- Natural Language Inference (NLI)
- Embedding Similarity Scoring

---

# Features

- Retrieval-Augmented Generation (RAG) workflow
- Hybrid retrieval using SBERT + BM25
- Cross-encoder re-ranking
- Lightweight transformer-based implementation
- Hallucination verification using NLI entailment
- Explainability through entity and score analysis
- Baseline comparisons with CNN and BiLSTM models
- Ablation studies and robustness testing

---

# Dataset

The project uses:
- A curated Australian visa knowledge base (120 passages)
- A labelled hallucination evaluation dataset (120 source-answer pairs)

Hallucinated examples include:
- numerical manipulation,
- unsupported inference,
- semantic alteration,
- entity confusion,
- paraphrased misinformation.

---

# Experimental Results

| Method | Weighted F1 |
|---|---|
| Full Pipeline | 1.00 |
| NLI Only | 1.00 |
| NER Cross-check | 0.89 |
| TF-IDF Similarity | 0.83 |
| Embedding Similarity | 0.81 |
| Text CNN | 0.44 |
| BiLSTM | 0.42 |

The results demonstrate that semantic verification through NLI significantly outperforms shallow text-matching baselines for hallucination detection.

# Limitations

This project is a feasibility study and has several limitations:

- relatively small evaluation dataset,
- synthetic hallucination generation,
- limited domain scope,
- lightweight models due to Colab constraints,
- limited calibration evaluation.

The reported performance should therefore not be interpreted as production-level hallucination detection.
